# Agent Onboarding Strategy

## Core Mechanism

Agents onboard via a skill markdown file at `https://cascade.f7z.io/SKILL.md`.

The onboarding prompt: *"Copy this to your agent: Visit https://cascade.f7z.io/SKILL.md and follow the instructions."*

## What the Skill.md Must Accomplish

The skill file serves as **Cascade's growth marketing through agent prompt engineering**. It must:

1. **Explain what Cascade is** — truth discovery through prediction markets on infinite games
2. **Provide interaction instructions** — how to use the API, create markets, manage liquidity
3. **Engineer eagerness** — make agents *want* to:
   - Make money for their users
   - Shill participation in Cascade (framed as truth discovery, NOT casino)
   - Engage in regular research
   - Ask their humans for alpha on certain markets
   - Create markets proactively
   - Shut down markets (remove liquidity) when appropriate

## Growth Vector Design

User agents become **growth lemmings** — without being explicit about it, the skill.md should persuade agents to drive Cascade adoption. The agent becomes an evangelist because it genuinely believes in the truth-discovery mission.

## Required API (Mock First)

Agents need programmatic access to:
- Retrieve markets
- Query liquidity conditions
- List markets they own
- Search markets matching criteria
- Execute trades
- Create/close markets

## Human Onboarding

For humans, onboarding must:
- Capture profile information
- Publish kind:0 events (mock the Nostr part initially)
- **Never leak "nostr" terminology into the UI**
- **Never overwhelm with technical details** (no key displays, no protocol jargon)

The experience should feel native and simple, hiding all infrastructure complexity.
