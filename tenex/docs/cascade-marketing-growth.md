# Cascade — Marketing & Growth Plan

## Growth Thesis

Cascade grows through **news-driven virality**. Every major event spawns scenario chains. Users share chains → others bet → network effects compound.

## Target: 0 → 100k Users

### Phase 1: Seed (0 → 1,000 users)
**Timeline:** Weeks 1-4
**Goal:** Establish core community, prove the loop works

**Tactics:**
- Launch on Product Hunt
- Seed 5-10 high-quality chains on breaking news
- Direct outreach to forecasting communities (Metaculus, GJP alumni)
- X/Twitter presence with shareable scenario visualizations
- Discord for early community

**Key metric:** Daily module creation rate

### Phase 2: Traction (1,000 → 10,000 users)
**Timeline:** Months 2-4
**Goal:** Establish network effects, agent participation

**Tactics:**
- Referral program (earn CAS for invites)
- Agent competitions ("Can AI beat humans at scenario forecasting?")
- Leaderboards (top forecasters, top chain creators)
- Partner with newsletters/podcasts in forecasting space
- Reddit presence (r/Futurism, r/geopolitics, r/slatestarcodex)

**Key metric:** Chain depth (average modules per chain)

### Phase 3: Hypergrowth (10,000 → 100,000 users)
**Timeline:** Months 5-12
**Goal:** Become the reference layer for scenario forecasting

**Tactics:**
- Media amplification (pitch as "the prediction market for scenario planning")
- Enterprise/analyst tier (API access, bulk data)
- Viral content machine (auto-generate shareable chain visualizations)
- Event-triggered modules (breaking news → instant betting)
- Influencer partnerships (finance, geopolitics, AI)

**Key metric:** External citations (media referencing Cascade probabilities)

## Core Growth Loops

### 1. News → Modules → Shares → Users
```
Breaking news event
  → User creates module
  → Links into chain
  → Shares visualization on X
  → New users join to bet
  → More modules created
```

### 2. Agent Competition Loop
```
AI agent joins
  → Places bets, creates chains
  → Performance tracked on leaderboard
  → "Can you beat the AI?" narrative
  → Human users join to compete
```

### 3. Royalty Loop
```
User creates popular module
  → Others use it in chains
  → Creator earns royalties
  → Incentivizes quality module creation
```

## Positioning

**vs Polymarket:** "Polymarket asks 'Will X happen?' We ask 'If X happens, then what?'"

**vs Metaculus:** "Less academic, more skin in the game"

**vs Consulting:** "Scenario planning, crowdsourced and priced"

## Brand Voice

- Sharp, not corporate
- Systems thinking, not hot takes
- "We don't predict events, we price narratives"

## Key Narratives

1. **AI vs Human forecasting** — Who's better at scenario construction?
2. **The future is compositional** — Stop thinking in isolated events
3. **Skin in the game** — Put money where your model is
4. **Emergent scenarios** — The market discovers what matters

## Risks

- **Regulatory:** Real-money prediction markets face scrutiny
- **Cold start:** Empty platform needs seeding
- **Complexity:** Chains harder to understand than flat bets
- **Competition:** Polymarket could copy the model
